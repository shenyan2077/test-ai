# 📘 前端AI编程能力考试

## 使用 Vue3 完整复刻一加 15T 官方产品页

---

## 一、考题目标

本考题旨在评估以下方面的综合能力：

- 页面结构与视觉还原能力
- 动画、滚动叙事与交互实现能力
- Vue3 技术栈掌握程度
- 前端工程化能力（组件化、目录结构、代码质量、构建交付）
- 对复杂产品官网页面的拆解、实现与调试能力
- 使用 AI 辅助开发的实际能力（通过提交聊天记录体现）

---

## 二、任务内容

### 1. 页面复刻

请使用 **Vue3 + Vite**，复刻以下页面的精简版：

**`https://www.oneplus.com/cn/15t`**

复刻对象为 **一加 15T 官方产品页**，以当前线上版本为准，但本题 **不要求复刻性能栏目和系统栏目**。

要求：

- 页面结构、布局、视觉效果尽可能做到 **高度还原**。
- 必须覆盖页面主要模块：顶部一加/OPPO 风格导航、一加 15T 产品导航、购买入口、Hero 首屏、设计与配色、1.1mm 极窄四等边、小屏手感、设计美图、屏幕、OPPO 自研 LUMO 凝光影像系统、7500mAh 冰川电池、耐用功能、备注与底部信息等。
- 顶部导航栏不需要下拉菜单、不需要二级页以及弹窗页。
- **不需要实现性能栏目**，包括但不限于第五代骁龙 8 至尊版、风驰游戏内核、游戏帧率、GPU 渲染超帧、跑分、热门手游适配等性能/游戏相关长篇内容；这些内容不作为验收和评分依据。
- **不需要实现系统栏目**，包括但不限于 ColorOS 16、流畅双引擎、AI 扫描、AI 一键闪记、游戏助手、跨生态互联等系统相关内容；这些内容不作为验收和评分依据。
- 页面需具备基本的 **响应式适配**（适配 1920px / 1024px / 390px）。
- 图片、视频、图标、SVG 等视觉资源必须来自原页面、一加/OPPO 官方公开资源或可核验的同源素材；不允许使用自制图片、AI 生成图片或通用占位图作为等效替代。
- 如原页面中的视频、滚动动画或图片序列在某个浏览器中无法完整查看，考生应更换浏览器或采用其他方式确认线上效果后再实现。


---

## 三、技术要求

- **必须使用 Vue3 + Vite**。
- **指定 Node.js 版本：22.12.10**。
- 必须使用组件化结构，合理拆分导航、Hero、产品卖点、设计配色、屏幕、小屏手感、影像、续航、参数卡片、页脚等模块。
- 样式可使用 SCSS / CSS Modules / Tailwind / 原生 CSS，需保证最终视觉统一、响应式良好。
- 动画实现方式不限，可使用 CSS、Web Animations API、IntersectionObserver、GSAP、Framer Motion 风格库或其他合理方案。
- 资源（图片、视频、SVG）应优先从原页面提取，或使用一加/OPPO 官方公开资源、可核验的同源素材；不允许使用自制图片、AI 生成图片或通用占位图作为等效替代。资源来源、提取方式和本地化处理需在 README 中说明。
- **不允许通过下载原页面 HTML / CSS / JS 后直接集成的方式实现**；不得将原站整页源码通过 `v-html`、`innerHTML`、`iframe` 或直接引入原站打包脚本等方式嵌入项目。页面结构、样式与交互应基于 Vue3 组件化重新实现。
- 项目需能通过 `npm install`、`npm run build` 生成可部署的 `dist` 产物。

---

## 四、提交内容

需提交一个压缩包。压缩包解压后，根目录必须分为以下三部分内容：

1. **源码**
   - 放置完整项目源码
   - 包含 `package.json`、入口文件、组件、样式、资源等
   - 包含开发说明文档 `README.md`
   - README 需说明：项目结构、组件拆分、主要动画与交互方案、响应式方案、资源来源与处理方式、AI 辅助生成页面的实现思路
1. **dist**
   - 放置执行 `npm run build` 后生成的可部署产物
   - dist 应为可直接部署到本地静态服务的完整构建结果
3. **AI记录**
   - 放置与 AI 的完整聊天记录（可为 `.md` 、`.txt` 、`.html` 格式文本）
   - 放置 AI 生成的相关文档

## 五、验收说明

1. 评分以考生提交的项目源码、dist 产物、README 和 AI 聊天记录为准，不以现场演示或口头说明作为主要依据。
2. 页面应在 Chrome 最新稳定版下正常渲染，核心视觉区块（导航栏、Hero、设计/配色区、小屏手感区、屏幕区、影像区、续航区、耐用功能区、底部备注）应完整可见且主要交互可用。
3. 性能/游戏及系统相关栏目不纳入验收范围；未实现第五代骁龙 8 至尊版、风驰游戏内核、游戏帧率、GPU 渲染超帧、跑分、ColorOS 16、AI 功能、游戏助手、跨生态互联等内容不扣分。
4. 响应式断点 1920px / 1024px / 390px 下布局不应严重错位、内容遮挡或关键信息丢失。
5. 动画效果（滚动触发、吸顶导航、图片/视频切换、hover 交互、模块进入动效）应流畅，无明显卡顿、闪烁、黑屏、空白或破图问题。
6. 源码不应采用“下载原页面 HTML / CSS / JS 后直接集成”的实现方式；如发现通过 `v-html`、`innerHTML`、`iframe` 或直接引入原站打包脚本来承载整页源码，视为不符合技术要求。
7. 如果使用视频或图片序列，应保证加载失败时有合理降级表现，不应出现大面积空白或破图。
8. AI 聊天记录需完整导出，与提交代码的对应关系可追溯，即能从聊天记录中找到考生是如何通过 AI 完成关键模块的。
9. 题目描述的是必须满足的结果；考生可自行选择合理的实现方案、动画库和资源处理方式。

## 六、统一验收运行口径

评分环境为 Windows、Node.js 22.12.10、Chrome 最新稳定版。评分人员按以下步骤验证：

1. 解压考生提交的压缩包，确认解压后的根目录包含且清晰分为 `源码`、`dist`、`AI记录` 三部分内容。
2. 在 `源码` 目录执行 `npm install` 安装依赖。
3. 在 `源码` 目录执行 `npm run build`，确认能正常生成构建产物；构建结果应与提交的 `dist` 目录内容一致或等效。
4. 将提交的 `dist` 目录部署到本地静态服务（nginx 或 `npx serve -s dist`）。
5. 在 Chrome 中打开页面，逐一检查：顶部导航 → 产品导航 → Hero 首屏 → 设计/配色 → 小屏手感 → 屏幕 → 影像 → 续航 → 耐用功能 → 备注与底部信息。
6. 验收时跳过性能/游戏及系统栏目，不要求检查第五代骁龙 8 至尊版、风驰游戏内核、游戏帧率、GPU 渲染超帧、跑分、ColorOS 16、AI 功能、游戏助手、跨生态互联等模块。
7. 分别以 1920px / 1024px / 390px 视口宽度检查响应式适配。
8. 验证滚动触发动画、吸顶导航、颜色/图片切换、视频或图片序列、hover 交互是否生效。
9. 打开 `源码/README.md` 和 `AI记录` 中的聊天记录，对照关键模块（如动画实现、组件拆分、资源提取、响应式处理）验证 AI 协作的有效性。
10. `dist` 应为可部署产物，不应依赖外部构建工具或需要评分人员手工修改文件。

## 七、评分说明

评分主要依据：

1. **视觉还原度** — 与一加 15T 官方页面精简范围内的布局、色彩、字体、间距、图片比例、内容节奏的接近程度。
2. **动画与交互质量** — 滚动叙事、吸顶导航、图片/视频切换、hover 效果、模块进入动效的流畅度和完整度。
3. **组件化与代码质量** — 组件拆分合理性、目录结构、命名规范、状态管理、可维护性。
4. **响应式适配** — 1920px / 1024px / 390px 三档断点下的布局表现、内容完整性和交互可用性。
5. **工程交付质量** — 源码与 dist 是否完整、README 完整性、构建稳定性、dist 可部署性。
6. **AI 协作质量** — 聊天记录完整性、AI 辅助关键决策与代码可追溯性、考生对 AI 输出的验证和修正能力。
